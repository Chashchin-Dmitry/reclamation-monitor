# Reclamation Monitor - Windows Service

## Быстрый старт

```cmd
# 1. Запустить интерактивное меню
menu.bat

# 2. Выбрать [6] для установки (требует Админа)

# 3. Готово! Сервис работает.
```

## Интерактивное меню

```cmd
menu.bat
```
Показывает статус сервиса и Ollama, все команды в одном месте.

## Команды управления

| Файл | Описание |
|------|----------|
| `menu.bat` | **Интерактивное меню** со статусом (рекомендуется) |
| `install.bat` | Установка сервиса (от Админа) |
| `start.bat` | Запуск сервиса |
| `stop.bat` | Остановка сервиса |
| `restart.bat` | Перезапуск (после git pull) |
| `status.bat` | Диагностика: сервис + Ollama + логи |
| `logs.bat` | Просмотр логов в реальном времени |
| `uninstall.bat` | Удаление сервиса (от Админа) |

## Структура логов

```
email_processor_project/
├── logs/
│   ├── service_stdout.log   # stdout сервиса
│   └── service_stderr.log   # stderr сервиса (основной)
├── email_processor.log      # лог приложения
└── processed_reclamations.json  # кэш обработанных писем
```

## Что делает сервис

1. Каждые 2 минуты проверяет почту (EMAIL_USER из .env)
2. Классифицирует письма по категориям (настраиваются в classifier_config.json)
3. Анализирует через LLM (Ollama + Qwen3)
4. Отправляет рекламации в Bitrix24
5. Пересылает ответственным лицам

## Зависимости

- **Ollama** должна быть запущена (порт 11434)
- **Модель**: qwen3:30b-a3b (или gpt-oss:20b)
- **Python venv** в директории проекта

## Автозапуск

Сервис настроен на `SERVICE_AUTO_START` — запускается автоматически при старте Windows.

## Обновление кода

```cmd
# 1. Остановить
stop.bat

# 2. Обновить код
git pull

# 3. Запустить
start.bat
```

## Диагностика проблем

### Сервис не запускается
```cmd
status.bat    # проверить статус
logs.bat      # посмотреть ошибки
```

### Ollama не работает
```cmd
# Проверить
curl http://localhost:11434/api/tags

# Запустить Ollama (обычно автозапуск)
ollama serve
```

### Письма не обрабатываются
1. Проверить `.env` (EMAIL_USER, IMAP_PASSWORD)
2. Проверить `processed_reclamations.json` (не закэшировано ли)
3. Смотреть `email_processor.log`

## Удаление

```cmd
uninstall.bat   # от Администратора
```

Сервис удаляется, файлы проекта и логи сохраняются.
